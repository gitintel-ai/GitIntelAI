# Security Policy

## Reporting a Vulnerability

Email **security@gitintel.ai** with:

- Description of the vulnerability
- Steps to reproduce
- Impact assessment (what an attacker could do)

**Do not** open a public GitHub issue for security vulnerabilities.

## Response Timeline

- **Acknowledge:** Within 48 hours
- **Triage:** Within 5 business days
- **Fix (critical):** Within 7 days
- **Fix (non-critical):** Next scheduled release

## Scope

- GitIntel CLI (`packages/cli/`)
- GitIntel Server (`packages/server/`)
- GitIntel Dashboard (`packages/dashboard/`)
- Attribution standard format

## Out of Scope

- Social engineering attacks
- Denial of service
- Issues in dependencies (report upstream, notify us)
- Attacks requiring physical access to the machine

## Recognition

We credit reporters in the CHANGELOG unless they prefer anonymity.
