# Spec 04: Context & Memory Manager

## Overview
Automatically generate, optimize, and manage CLAUDE.md context files to minimize
token usage while maximizing AI coding quality.

## gitintel context init

### Repo Scanner
Detects:
- **Language/Framework**: package.json → Next.js/React/Vue; Cargo.toml → Rust; requirements.txt → Python
- **Test framework**: jest, vitest, pytest, cargo test
- **Linting**: .eslintrc, .prettierrc, biome.json, rustfmt.toml
- **Commit convention**: .commitlintrc, conventional commits patterns in history
- **Architecture**: src/ structure, domain boundaries, key entry points
- **Dependencies**: top 10 by usage frequency in imports

## gitintel context optimize

### Section Scoring Algorithm
```
score(section) = (reference_count / total_sessions) × recency_weight

Where:
- reference_count = number of recent sessions where AI referenced this section
- total_sessions = sessions analyzed (default: last 30)
- recency_weight = exponential decay: 1.0 for today, 0.5 for 7d ago, 0.1 for 30d ago

Pruning rules:
- score < 0.05 → PRUNE (safe to remove)
- score < 0.20 → COMPRESS (summarize to 1-2 lines)
- score >= 0.20 → KEEP (as-is)
```

## Memory Store

### Commands
```bash
# Add a fact to memory
gitintel memory add --key "auth-pattern" \
  --value "All auth uses JWT via middleware in src/middleware/auth.ts" \
  --category "architecture"

# List memory by category
gitintel memory list --category architecture

# Show memory stats
gitintel memory stats
# → 24 facts | 1,847 tokens | 12 used in last 7 days

# Prune unused facts
gitintel memory prune --unused-days 30

# Export memory as CLAUDE.md section
gitintel memory export >> CLAUDE.md
```

### Auto-Memory Generation
After each session (tracked via OTel), gitintel:
1. Analyzes what the AI asked about repeatedly
2. Extracts recurring patterns as memory facts
3. Proposes facts to the developer for approval
4. Adds approved facts to the memory store

### MCP Server Integration
```json
// .claude/settings.json — add gitintel as MCP memory server
{
  "mcpServers": {
    "gitintel-memory": {
      "command": "gitintel",
      "args": ["mcp", "serve"],
      "description": "GitIntel AI project memory store"
    }
  }
}
```

This exposes memory as MCP tools: `read_memory`, `write_memory`, `list_memory`
so Claude Code can directly query/update the memory store during sessions.
